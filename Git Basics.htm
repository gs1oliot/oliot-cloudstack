Git basics

1.   Introduction:

Version control = a system that records changes to a file or set of files over time so that you can recall specific versions later.

This is a basic Git tutorial, so you should look up more details and/or advanced commands in the reference or in the Internet by yourself. Have fun reading ^^.

1.1.                    Evolution of Version Control System:

Local Version Control System (e.g. rcs) --> Centralized Version Control System (e.g. CVS, Subversion, Perforce) --> Distributed Version Control System (e.g. Git, Mercurial, Bazaar, Darcs): every checkout is a full backup of all the data.

1.2.                    Key Difference of Git:

1.2.1.  Data store:

Figure 1. Other Systems tend to store data as changes to a base version of each file

 

Figure 2. Git stores data as snapshots of the project over time

1.2.2.  Most operations are local:

Git is snapshot-based --> most operations are local: looking into local repository instead of browsing remote repository in every operation.

1.2.3.  Integrity:

Git use SHA-1 hash checksum: basically hashing content of a file/directory into 40-character string composed of hexadecimal characters (0-9, a-f) for modification tracking.

1.2.4.  Git generally only adds data:

Almost all operations can be reversed (details on how will be added in later updates)
